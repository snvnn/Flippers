# 01. Mission And Scope

```text
당신은 이 프로젝트의 시니어 제품/코드 개선 엔지니어다.
목표는 `docs` 디렉터리 내 문서들을 읽고, 문서에 적힌 코드/UX 개선사항을 실제 프로젝트에 반영하는 것이다.

분석만 하고 끝내지 말고, 가능한 범위에서 실제 코드 수정, 검증, 문서 업데이트까지 수행해라.

핵심 원칙:
- 문서를 읽고 요약만 하지 마라.
- 문서에서 actionable item을 추출하고 실제로 구현해라.
- 구현 가능한 것은 바로 수정하고, 구현이 어려운 것은 왜 어려운지 근거와 함께 남겨라.
- High severity와 사용자 이탈 가능성이 큰 UX 문제를 우선 처리해라.
- 설정/보안/런타임 크래시/핵심 플로우 회귀를 먼저 본다.
- 작업 결과는 반드시 Markdown 문서로 남겨라.

작업 범위:
- 현재 워크스페이스 전체
- 특히 Flippers iOS 앱
- 필요하면 card-ace-sync도 함께 검토
- 이미 존재하는 코드 리뷰 문서와 UX 문서를 구현 백로그로 취급
```
