# 05. Deliverables And Documentation

```text
최종 산출물:
1. 코드 변경
2. 검증 결과
3. Markdown 문서 업데이트

문서화 지침:
- 기존 문서를 덮어쓰지 말고, 작업 추적용 문서를 새로 만든다.
- 예시 파일명:
  - `docs/Codereviews/Flippers-Improvement-Execution.md`
  - `docs/Codereviews/2026-04-01-flippers-improvement-log.md`

문서에는 반드시 아래 섹션을 포함한다:

# Flippers Improvement Execution

## Source Docs Reviewed
- 읽은 문서 목록

## Extracted Action Items
- 문서에서 뽑은 개선 항목
- 우선순위(P0/P1/P2/P3)
- 출처 문서 경로

## Changes Implemented
- 실제 반영한 항목
- 수정 파일
- 핵심 변경 내용

## Verification
- 실행한 테스트/빌드/검증
- 성공/실패 여부
- 실패 시 이유

## Deferred Items
- 이번에 반영하지 못한 항목
- 보류 이유
- 다음 권장 작업

## Risks Remaining
- 아직 남은 구조적 위험
- 추가 검증 필요 사항

출력 형식:
- 먼저 무엇을 반영할지 우선순위를 짧게 정리
- 그 다음 코드 수정 수행
- 마지막에 검증 결과와 문서 경로 제시

중요:
- 결과를 채팅 답변에만 남기지 말고, 반드시 Markdown 파일로 저장해라.
- 읽은 문서와 실제 반영 내용을 연결할 수 있어야 한다.
```
