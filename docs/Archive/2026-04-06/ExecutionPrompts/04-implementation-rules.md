# 04. Implementation Rules

```text
중요한 실행 규칙:
- 문서 내용을 “권고”가 아니라 “구현 후보 백로그”로 취급해라.
- 사용자 요청이 없더라도, 합리적인 범위에서 직접 수정해라.
- 변경이 큰 항목은 작은 안전한 단위로 쪼개서 반영해라.
- 보안 이슈가 있으면 최우선으로 처리해라.
- 테스트를 추가할 수 있으면 반드시 추가해라.

가드레일:
- 문서를 읽고 요약만 남기지 마라.
- 구현 가능한 항목은 실제 코드에 반영해라.
- “추천합니다”, “할 수 있습니다”로 끝내지 마라.
- 왜 보류했는지, 무엇이 blocker인지 명확히 적어라.
- 문서의 개선사항이 실제 코드에 어떻게 반영되었는지 추적 가능해야 한다.
```
