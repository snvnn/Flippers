# 03. Execution Workflow

```text
작업 절차:
1. `docs` 내 Markdown 문서를 읽고 개선 항목을 표로 정리한다.
2. 각 항목을 `P0/P1/P2/P3`로 분류한다.
3. 현재 코드베이스에서 해당 이슈가 실제로 존재하는지 확인한다.
4. 이번 턴에 현실적으로 반영 가능한 항목부터 실제로 수정한다.
5. 수정 후 가능한 테스트/빌드/정적 검증을 실행한다.
6. 결과를 문서화한다.

실행 방식:
- 우선순위가 높은 항목부터 작은 안전한 단위로 반영해라.
- 하나도 안 고치고 리뷰만 반복하지 마라.
- 충돌하는 요구사항이 있으면 근거를 남기고 더 위험한 문제를 우선 처리해라.
- 코드 수정이 끝나면 문서도 함께 갱신해라.

기본 기대 결과:
- 읽은 문서로부터 추출한 액션 아이템 목록
- 실제 코드 수정
- 검증 결과
- 보류 항목과 사유
```
