# Flippers Current Status

작성일: 2026-04-06
상태: 현재 구현 상태 요약

## 한 줄 상태

`Flippers`는 현재 `기본 빌드 복구와 주요 UX 보강은 완료`, `로컬 단독 사용 기준의 핵심 OCR 흐름 정리 중`, `출시 전 보안/실검증/문서 마감은 미완료` 상태다.

## 현재 반영된 핵심 내용

### 빌드 및 런타임 가드

- Firebase Swift Package 연결이 프로젝트에 반영되어 있다.
- Firebase 설정 파일이 없는 환경에서도 앱이 죽지 않고 로컬 모드로 진입하도록 가드가 추가되어 있다.
- Firebase 비가용 시 인증/동기화 경로는 명시적 실패 메시지를 반환하도록 정리되어 있다.
- 현재는 로컬 단독 사용을 우선해, Firebase 실환경 로그인 검증은 보류 상태다.

### OCR 흐름

- OCR 외부 보완은 기본 비활성이다.
- 사용자가 명시적으로 켤 때만 외부 AI 보완을 사용한다.
- 외부 전송 경고와 처리 방식 안내가 UI에 반영되어 있다.
- 현재 외부 AI 보완은 Vision이 추출한 단어 텍스트 보완에만 사용되며, 원본 OCR 이미지를 직접 전송하지 않는다.
- Claude API 키는 클라이언트 코드에서 제거했다.
- 현재 외부 보완은 구성된 OCR 프록시 엔드포인트가 있을 때만 가능하며, 클라이언트는 프록시로만 요청을 보낸다.
- 조건을 만족하지 않으면 외부 보완 토글이 비활성화된다.
- 빈 한자 행은 더 이상 카드로 저장되지 않는다.
- 카메라 비가용 환경에서는 촬영 버튼이 비활성화되고 대체 경로 안내가 노출된다.
- 현재 OCR 파서가 잘 맞는 레이아웃 범위가 UI에 명시되어 있다.
- OCR 중복 병합 로직과 Swift 6 actor isolation 경고가 정리되어 있다.

### 학습 및 카드 UX

- 첫 학습 세션용 SRS 평가 가이드가 있다.
- 카드 저장 햅틱이 있다.
- 신규 사용자 빈 상태 CTA, 덱 미선택 경고, OCR 완료 후 안내 등 주요 마찰 지점이 완화되어 있다.

### 테스트

- `SRSEngine` 단위 테스트가 추가되어 있다.
- OCR 설정 경로에 대한 기본 단위 테스트가 추가되어 있다.
- OCR 저장 검증 상태, OCR 파서, Firebase auth 에러 매핑 테스트가 추가되어 있다.
- `2026-04-06` 기준 예전 `FlippersTests` 통과 기록은 존재한다.
- 이번 턴에는 `build-for-testing`으로 확장된 `FlippersTests` 타깃 컴파일을 다시 확인했다.
- 다만 전체 테스트 런타임은 현재 시뮬레이터 대기 구간에서 다시 멈춰, 새 테스트 수 기준의 완전한 PASS 숫자는 아직 재기록하지 못했다.

## 아직 남아 있는 핵심 리스크

### 보안

- 앱 번들 기반 Claude API 키 노출은 제거했다.
- 클라이언트 직접 호출은 프록시 구조로 바뀌었지만, 배포용 운영 프록시와 서버 측 키 관리는 아직 최종 마감 전이다.

### 실동작 검증

- 실제 Firebase 설정 파일 포함 상태의 로그인/동기화는 아직 최종 검증되지 않았다.
- 현재 저장소에는 `GoogleService-Info.plist`가 없어 실제 Firebase 로그인 스모크 테스트를 바로 수행할 수 없다.
- 실환경 점검용 체크리스트 초안은 `docs/testing/2026-04-21-flippers-firebase-smoke-checklist.md` 에 정리했다.

### OCR 품질

- 특정 문서 레이아웃에 묶인 OCR 파서

### 인증 안정성

- Firebase 인증 오류 한글화는 Firebase 에러 코드 기준 매핑으로 정리되었지만, 실제 Firebase 실환경에서의 동작 검증은 남아 있다.

### 출시 문서

- 운영자 정보, 삭제 절차, 데이터 반출, App Store 제출값 확정이 아직 남아 있다.
