# Flippers Release Privacy Summary

작성일: 2026-04-06
상태: 현재 출시 판단용 압축 문서

## 목적

긴 컴플라이언스 초안과 정책 초안을 매번 열지 않아도, 현재 출시 전 확인해야 할 프라이버시 범위를 빠르게 파악할 수 있게 정리한 문서다.

## 현재 다루는 주요 데이터

- 이메일 주소
- 사용자 식별자
- 카드, 덱, 섹션, SRS 상태, 학습 로그
- OCR 이미지
- OCR 추출 텍스트

## 현재 관여하는 외부 처리 경로

### Apple

- Sign in with Apple
- CloudKit 자동 동기화
- 사진 선택 및 카메라 권한

### Firebase

- 인증
- 동기화 관련 서비스 코드 존재

### Anthropic Claude API

- 현재는 사용자가 OCR 외부 보완을 켠 경우에만 Vision이 추출한 텍스트가 OCR 프록시 서버로 전송될 수 있고, 서버가 Claude API에 전달할 수 있다.
- 원본 OCR 이미지는 앱에서 직접 Anthropic으로 전송하지 않는다.
- 클라이언트 코드에는 Claude API 키가 없고, 직접 Claude 호출도 없다.
- 현재 OCR 프록시 운영 계약은 `text-only enhancement` 기준이며, 이미지 직접 업로드 payload는 허용하지 않는다.

## 현재 기준 출시 전 필수 확인 항목

### 1. 보안 구조

- Claude API 키를 클라이언트 번들에서 제거하는 작업은 완료했다.
- 프록시 호출 구조 전환은 반영했다.
- 프록시 `text-only` 계약과 운영 runbook 초안은 `docs/operations/2026-04-21-flippers-ocr-proxy-runbook.md` 에 정리했다.
- 다음 단계로 실제 운영 프록시 배포와 서버 측 키 관리가 필요하다.
- 실환경 검증은 Release / Archive 배포용 빌드 + 실제 Firebase 프로젝트 + 실제 Apple ID 기준으로만 PASS 처리하고, 사용한 빌드 ID 를 함께 기록한다.
- Firebase smoke 와 Apple Sign-In smoke 는 같은 빌드 ID 의 증거여야 한다.
- 시뮬레이터 / 로컬 목 / 개발용 fallback 은 참고용이며 PASS 근거가 아니다.
- 과거에 노출된 기존 키가 있다면 별도로 폐기와 재발급이 필요하다.

### 2. 공개 문서

- 개인정보처리방침 초안 마감
- App Store Privacy 제출값 확정
- 이용약관은 이번 범위에서 새로 작성하지 않고 별도 결정으로 둔다.
- 현재 비배포 전제 기준의 정리용 체크리스트는 `docs/release/2026-04-21-flippers-release-readiness-checklist.md` 에 정리했다.

### 3. 사용자 권리 절차

- 계정 삭제 또는 전체 데이터 삭제 경로
- 문의/요청 접수 경로
- 필요 시 데이터 반출 정책
- 현재 확보된 운영 정보는 운영자 `윤현`, 문의 이메일 `yunh1205@gmail.com` 이고, 삭제/반출 절차는 아직 미정이다.
- 가장 안전한 기본값은 `yunh1205@gmail.com` 을 공개 문의와 개인정보/권리 요청의 단일 접수 창구로 쓰는 것이다.
- 계정 삭제와 전체 데이터 삭제는 같은 이메일 기반 수동 처리 경로로 묶는다.
- 데이터 반출(export)은 기본 미지원으로 두고, 요청이 오면 미지원이라고 안내한다.
- App Store 제출이나 공개 문구에서도 이 단일 이메일을 기본 지원 접점으로 맞춘다.
- URL이 강제될 때만 같은 이메일만 안내하는 정보 제공 전용 정적 페이지를 사용하고, 그 페이지에는 폼/추적/추가 수집을 넣지 않는다.

### 4. 고지와 선택권

- OCR 외부 전송 고지 유지
- opt-in 유지
- 민감정보 업로드 지양 안내 유지

## 현재 판단

- OCR 외부 전송 고지와 선택권은 1차 반영되었다.
- 공개 문서 초안도 생겼다.
- 번들 기반 Claude 키 노출도 제거했다.
- 하지만 출시 판단 기준으로는 아직 `보안 구조`와 `운영 절차`가 덜 끝난 상태다.
