# Flippers Current Priorities

작성일: 2026-04-06
상태: 현재 작업 우선순위 기준 문서

## 목적

이 문서는 현재 코드와 최근 작업 결과를 기준으로, 지금 바로 처리해야 할 순서를 짧게 고정한다.

## 현재 최우선 목표

`Flippers`를 출시 가능한 수준으로 안정화하는 것이 현재 최우선이다.  
핵심 기준은 `보안`, `실동작 검증`, `핵심 OCR/인증 플로우 안정화`다.

현재는 `로컬 단독 사용`을 우선하므로 Firebase 로그인 실환경 검증은 잠시 뒤로 미루고, OCR 정확도와 로컬 사용 흐름 안정화에 더 높은 비중을 둔다.

## 우선순위

### 1. P0 — Claude OCR 운영 경로 정리

- `Config.plist` 번들 기반 키 노출은 제거했다.
- 현재 iOS 클라이언트의 직접 Claude 호출은 제거했고, OCR 외부 보완은 프록시 엔드포인트를 통해서만 전달된다.
- 현재 앱에서 Claude는 Vision이 추출한 단어 텍스트 보완에만 사용되며, 원본 OCR 이미지를 직접 업로드하지 않는다.
- 로컬 mock 프록시와 운영 프록시 모두 `text-only enhancement` 계약만 허용하도록 정리했고, 레거시 이미지 직접 업로드 payload는 더 이상 운영 계약에 포함하지 않는다.
- 현재 저장소에는 로컬 mock 프록시 스텁이 포함되어 있으며, 출시 전에는 실제 운영 프록시 배포와 서버 측 키 관리가 필요하다.
- 운영 기준과 배포 체크리스트는 `docs/operations/2026-04-21-flippers-ocr-proxy-runbook.md` 에 정리했다.
- 기존에 노출된 Claude 키가 있다면 즉시 폐기하고 새 키로 교체해야 한다.
- 종료 조건:
  - 배포 빌드에서 앱 번들에 실제 비밀 키가 포함되지 않는다.
  - 텍스트 보완 외부 호출 경로가 운영 프록시로 연결되고 서버 측 키 관리가 정리된다.

### 2. P1 — OCR 플로우 안정화

- 현재 남아 있는 대표 이슈:
  - 특정 레이아웃에 강하게 묶인 파서
- 이번 턴에 빈 카드 생성 방지, 카메라 비가용 안내, 지원 레이아웃 명시는 반영했다.
- 종료 조건:
  - OCR 지원 범위 명시 또는 파서 개선

### 3. P1-P2 — 인증 오류 처리 안정화

- Firebase 오류 한글화는 Firebase 에러 코드 기준 매핑으로 전환했다.
- 로컬 단독 사용 중에는 우선순위가 낮지만, 이후 로그인 기능을 다시 켤 때 실제 Firebase 실환경에서 이 매핑을 확인해야 한다.
- 종료 조건:
  - 실제 로그인/회원가입/Apple 로그인에서 대표 오류 케이스 확인

### 4. P2 — Firebase 실환경 검증

- 빌드 복구와 로컬 모드 fallback은 반영됐지만, 실제 Firebase 설정 파일 포함 상태에서 로그인과 동기화는 아직 검증되지 않았다.
- Apple Sign In / CloudKit entitlements 뼈대는 프로젝트에 연결했다.
- 다만 현재 저장소에는 `GoogleService-Info.plist`가 없어 실제 Firebase 로그인 스모크 테스트를 바로 진행할 수 없다.
- 실환경 점검 순서는 `docs/testing/2026-04-21-flippers-firebase-smoke-checklist.md` 에 정리했다.
- 종료 조건:
  - `GoogleService-Info.plist` 포함 상태에서 앱 실행 확인
  - 이메일 로그인 확인
  - Apple 로그인 확인
  - Firestore 또는 실제 사용하는 동기화 경로 확인

### 5. P2 — 테스트 보강

- 현재 의미 있는 자동 테스트는 `SRSEngine`, OCR 설정, OCR 저장 상태, OCR 파서, auth 에러 매핑까지 확대되었다.
- OCR, auth, 로컬 모드, Firebase 비가용 처리 회귀 방어가 약하다.
- 종료 조건:
  - OCR 파서 회귀 검증 유지
  - Firebase 비가용 진입 경로 검증

### 6. P2-P3 — 출시 문서 마감

- 공개용 개인정보처리방침, App Store Privacy, 삭제/반출 절차, 운영자 정보 반영이 아직 마감 전 상태다.
- 종료 조건:
  - 운영자 정보 반영
  - 삭제/문의 절차 반영
  - App Store 제출값 확정
  - 필요 시 이용약관 초안 추가

## 이미 완료된 축

- Firebase SPM 연결 및 기본 빌드 복구
- Firebase 미설정 시 로컬 모드 진입
- OCR 외부 전송 고지 및 opt-in
- Claude API 키의 번들 `Config.plist` 의존 제거
- 클라이언트 직접 Claude 호출 제거 및 프록시 호출 구조 전환
- OCR 프록시 `text-only` 계약 고정 및 운영 runbook 초안 추가
- Apple Sign In / CloudKit entitlements 뼈대 연결
- 빈 카드 생성 방지, 카메라 비가용 안내, OCR 지원 범위 명시
- Firebase 인증 오류의 에러 코드 기반 매핑 전환
- Swift 6 actor isolation 경고 정리
- `SRSEngine` 단위 테스트 추가
